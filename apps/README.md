# Hardened app build recipes

Each app this clone packages lives in its own folder here so the per-app config
does not drift through the shared files in `src-tauri/`. One clone, no branch
switching.

## How it works

`apps/build.sh <app>` does the following:

1. Sources `apps/<app>/manifest.sh` for the URL, name, identifier, icon, size,
   and any extra CLI flags.
2. Overlays the app's config into `src-tauri/`:
   - `apps/<app>/Info.plist` → `src-tauri/Info.plist` (falls back to `_base/Info.plist`)
   - `apps/<app>/capabilities.json` → `src-tauri/capabilities/default.json` (falls back to `_base/capabilities.json`)
3. Runs the Pake CLI (`node dist/cli.js`) with the manifest's flags.
   `entitlements.plist` is regenerated by the CLI from `--camera` / `--microphone`.
4. Moves the built `.app` (and `--keep-binary` output) into `apps/<app>/output/`.
5. Restores `src-tauri/{Info.plist,capabilities/default.json,entitlements.plist}`
   from `_base/` so the working tree stays clean between builds.

## Which files are per-app vs shared

| File | Handling | Owner |
| --- | --- | --- |
| `src-tauri/Info.plist` | static, read verbatim | per-app overlay or `_base` |
| `src-tauri/capabilities/default.json` | static, read verbatim | per-app overlay or `_base` |
| `src-tauri/entitlements.plist` | regenerated from `--camera`/`--microphone` | manifest flags |
| `src-tauri/.pake/`, `tauri.conf.json` merge | regenerated per build | base config + flags |

`_base/` holds the generic hardened defaults (no camera/mic usage strings, plain
`capabilities` with no remote IPC, empty entitlements). Anything an app needs
beyond that lives in the app's own folder.

## Add a new app

```
apps/
  <app>/
    manifest.sh        # required
    Info.plist         # optional; only if the app needs usage strings
    capabilities.json  # optional; only if the app needs a remote/IPC capability
```

## Usage

```bash
bash apps/build.sh spotify
bash apps/build.sh google-meet
```
